# Mission

## What co is

**Your trusted local personal intelligence company — a partner and assistant agent.**

*The best of co makes a better of you.*

- **Trusted** — explicit approval boundary, inspectable state, reversible actions, grounded output
- **Local** — user-controlled files, memory, configuration, integrations, and history
- **Personal** — durable user model: preferences, habits, projects, relationships, working context; the system's model of the user is the user's property — viewable, correctable, and fully owned
- **Intelligence partner** — grows together with the user: co deepens its understanding of the user while helping the user think, decide, and work better over time
- **Unbounded scope** — not limited to one domain; operates across work, research, planning, health, finance, creative, and any area where intelligence and continuity matter

## Ultimate goal

A persistent personal intelligence layer that can:

1. Understand the user, their projects, and their norms
2. Combine local, private, and web context into one working model
3. Execute bounded actions asynchronously and safely
4. Maintain continuity across sessions, devices, and time
5. Grow together with the user — every interaction deepens co's understanding and sharpens the user's judgment; value compounds across sessions, it does not reset

When co reaches that point, terminal is the control surface — not the product boundary.

## Strategic thesis

The design lineage is Engelbart's: augment human intellect, not replace it.

co should not try to beat cloud products by being broader. It should beat them by being:

- More inspectable
- More user-owned
- More composable — integrations, tools, and memory are user-assembled, not locked-in
- More project-aware
- More privacy-preserving
- More faithful to the user's real working context

Strategy: **local memory + bounded autonomy + explicit user control** — not full-auto agent demos.

## Stages

| Stage | Goal | Focus |
|---|---|---|
| 1 — Reliability | Safe and dependable enough for longer workflows | Action previews, approval ergonomics, task observability, compaction quality |
| 2 — Personalization | From memory recall to a durable user model | User profile, project norms, preference/habit representations, inspection surfaces |
| 3 — Bounded autonomy | Own longer-lived work without pretending to act unsupervised | Recurring tasks, plan execution with approval checkpoints, follow-up behaviors, scoped policy modes |
| 4 — Cross-surface continuity | Persistent intelligence partner across contexts, not just one terminal session | Voice/notification surfaces, cross-device continuity, multimodal memory inputs |
| 5 — Personal OS layer | User's inspectable intelligence layer across tools, memory, and ongoing work | User-owned knowledge graph, agent-managed but user-auditable workflows, local policy as first-class primitive |

## Non-goals

- Chasing generic IDE-agent parity as the main strategy
- Overbuilding multi-agent systems before single-agent continuity is excellent
- Making memory or agent state opaque — the user's model is always inspectable and correctable
- Prioritizing theatrical persona behavior over completion quality
- Absorbing the user's judgment rather than sharpening it — the goal is a more capable thinker, not a more dependent one
- Adopting heavy infrastructure that breaks the local-first advantage without a clear win
