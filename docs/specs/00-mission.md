# Mission

## What co is

**A trusted local personal operator for knowledge work.**

- **Trusted** — explicit approval boundary, inspectable state, reversible actions, grounded output
- **Local** — user-controlled files, memory, configuration, integrations, and history
- **Personal** — durable user model: preferences, habits, projects, relationships, working context
- **Operator** — able to research, plan, execute, monitor, and follow up across tools
- **For knowledge work** — optimized for information synthesis, coordination, writing, planning, and technical execution; not only coding

## Ultimate goal

A persistent personal operating layer that can:

1. Understand the user, their projects, and their norms
2. Combine local, private, and web context into one working model
3. Execute bounded actions asynchronously and safely
4. Maintain continuity across sessions, devices, and time
5. Become more useful through long-term adaptation rather than prompt growth

When co reaches that point, terminal is the control surface — not the product boundary.

## Strategic thesis

co should not try to beat cloud products by being broader. It should beat them by being:

- More inspectable
- More user-owned
- More composable
- More project-aware
- More privacy-preserving
- More faithful to the user's real working context

Strategy: **local memory + bounded autonomy + explicit operator control** — not full-auto agent demos.

## Stages

| Stage | Goal | Focus |
|---|---|---|
| 1 — Reliability | Safe and dependable enough for longer workflows | Action previews, approval ergonomics, task observability, compaction quality |
| 2 — Personalization | From memory recall to a durable user model | User profile, project norms, preference/habit representations, inspection surfaces |
| 3 — Bounded autonomy | Own longer-lived work without pretending to act unsupervised | Recurring tasks, plan execution with approval checkpoints, follow-up behaviors, scoped policy modes |
| 4 — Cross-surface continuity | Persistent operator across contexts, not just one terminal session | Voice/notification surfaces, cross-device continuity, multimodal memory inputs |
| 5 — Personal OS layer | User's inspectable orchestration layer across tools, memory, and ongoing work | User-owned knowledge graph, agent-managed but user-auditable workflows, local policy as first-class primitive |

## Non-goals

- Chasing generic IDE-agent parity as the main strategy
- Overbuilding multi-agent systems before single-agent continuity is excellent
- Turning memory into opaque autonomous state the user cannot inspect or correct
- Prioritizing theatrical persona behavior over completion quality
- Adopting heavy infrastructure that breaks the local-first advantage without a clear win
