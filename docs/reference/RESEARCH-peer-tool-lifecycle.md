# RESEARCH: Peer Tool Lifecycle — Five-Way Comparison

Sources: `~/workspace_genai/fork-claude-code` (Anthropic Claude Code), `~/workspace_genai/gemini-cli` (Google Gemini CLI), `~/workspace_genai/codex` (OpenAI Codex CLI), `~/workspace_genai/opencode` (Open-source code assistant), co-cli current codebase
Scan date: 2026-04-03 (fork-cc, gemini-cli), 2026-04-04 (codex, opencode), 2026-04-05 (deep verification pass, co-cli update), 2026-04-06 (line-number + accuracy sync against latest co-cli code)

## Purpose

Five-way comparison of the complete tool lifecycle — from definition to model context — to identify structural gaps in co-cli's tooling implementation. Each claim is verified against source code. §1 is the raw comparison; §2–4 are source-verified gap analysis with corrections from deep code inspection.

## 3. Root Cause Analysis

Foundation shipped (commit 415a233): `tool_output()` has `ctx` parameter, shell migrated. Persistence and per-tool-type compaction shipped (commit 6191bfd): `persist_if_oversized()` at return time, `truncate_tool_results` refactored to per-tool-type recency clearing, `ToolResult` TypedDict replaced with pydantic-ai native `ToolReturn(return_value, metadata)` — `_kind` discriminator removed, display/model separation via SDK-native `return_value`/`metadata` split.
