# RESEARCH: Peer Tool Lifecycle — Five-Way Comparison

Sources: `~/workspace_genai/fork-claude-code` (Anthropic Claude Code), `~/workspace_genai/gemini-cli` (Google Gemini CLI), `~/workspace_genai/codex` (OpenAI Codex CLI), `~/workspace_genai/opencode` (Open-source code assistant), co-cli current codebase
Scan date: 2026-04-03 (fork-cc, gemini-cli, co-cli), 2026-04-04 (codex, opencode), 2026-04-05 (deep verification pass)

## Purpose

Five-way comparison of the complete tool lifecycle — from definition to model context — to identify structural gaps in co-cli's tooling implementation. Each claim is verified against source code. §1 is the raw comparison; §2–4 are source-verified gap analysis with corrections from deep code inspection.

## 3. Root Cause Analysis

Foundation shipped (commit 415a233): `tool_output()` has `ctx` parameter, shell migrated, `_kind` discriminator added.

| RCA | co-cli file | Observed gap | Peer convergence |
|-----|------------|-------------|-----------------|
| **`tool_output()` was a dict factory** | `tool_output.py:32–45` | Now has return-time persistence (lines 42–44): when `ctx is not None` and `len(display) > TOOL_RESULT_MAX_SIZE`, calls `persist_if_oversized()`. 5 compactable call sites pass `ctx=ctx`; ~74 others unchanged. `truncate_tool_returns` (`_history.py:180–239`) still handles per-turn compaction | 3/4 peers enforce size at/before gateway (fork-cc `processToolResultBlock`, gemini-cli `distill()`, opencode `Tool.define()` wrapper). Codex at construction time |
| **Single `display` field** | `tool_output.py:21–29` | One field serves terminal, model, and transcript. No way to give model condensed content while showing user formatted output | 2/4 separate (fork-cc method-based, gemini-cli dual fields). 2/4 don't (codex, opencode) |
| **No tool execution hooks** | N/A — no hook infrastructure | No interception beyond `ApprovalRequired`. No input modification, output enrichment, or plugin integration | 3/4 user-extensible (fork-cc plugins, gemini-cli config/ext, opencode plugins). Codex internal-only |

**In progress**: RCA-1 addressed by persistence inside `tool_output()` (`TODO-tool-output-compaction.md` TASK-1+2) for 6 compactable tools. RCA-2 and RCA-3 not addressed — future items.
