## 2. Storage Format

Three semantic types: `SEMANTIC`, `EPISODIC`, `PROCEDURAL` (`configs/enums.py`) | Vector-embedded facts in configurable store (23 backends via `VectorStoreFactory`: Qdrant, Chroma, PGVector, Milvus, Pinecone, MongoDB, Redis, Elasticsearch, FAISS, Weaviate, etc.) + optional Neo4j graph | Vector store + SQLite history + optional Neo4j |

Per-file frontmatter: `name`, `description`, `type` (user/feedback/project/reference) (`memoryTypes.ts:261–271`) | Same format — archival is just older entries in the same files | Filesystem (`.claude/agent-memory/`) + JSON snapshots |
| **openclaw** | Per-agent SQLite stores at `~/.local/share/openclaw/memory/{agentId}.sqlite`. Chunks with embeddings and line ranges. | **Read** | Block content included in every prompt (in-context) | `search(query, limit, filters, threshold)` — semantic similarity | FileRead on `MEMORY.md` + topic files | `recall_memory` tool (`memory.py:358`) → FTS5/BM25 search |
| **Update** | `memory(state, "str_replace", path, old_string, new_string)` | `update(memory_id, data, metadata)` — LLM decides merge | FileWrite/Edit to existing file | `update_memory(slug, old_content, new_content)` → exact str_replace in `.md` file + FTS re-index (`memory.py:830–941`). Full audit trail | `history(memory_id)` → SQLite `history` table with `old_memory`, `new_memory`, `event` | Snapshot `updatedAt` timestamps | No changelog or `history()` API. ## 4. Memory Lifecycle

| Aspect | letta | mem0 | fork-cc | co-cli |
|--------|-------|------|---------|--------|
| **Consolidation** | Not observed — archival passages are permanent | Not observed — version history via SQLite but no auto-consolidation | **autoDream**: LLM consolidation pass. Triggers: 24h + 5 sessions. Phases: orient → gather signal → consolidate → prune. Keeps index under 25KB, ~150 chars/entry. Forked subagent with read-only bash (`consolidationPrompt.ts:10–65`). **openclaw**: three-tier dreaming — Light (6h, dedup 0.9 threshold), Deep (daily, recency decay 14-day half-life, auto-recovery below 0.35 health), REM (weekly, pattern recognition 0.75 min strength) (`dreaming.ts`) | **Write-time agent-based upsert** via `check_and_save()` (`_save.py:64–100`). Triggered inside `persist_memory()` (`_lifecycle.py:27–65`). Builds manifest of existing memories (up to 100), sends candidate + manifest to save agent. Returns `SaveResult` with `action: "SAVE_NEW" | "UPDATE"` and optional `target_slug`. On UPDATE: `overwrite_memory()` (`_save.py:103–192`) merges tags, refreshes provenance/certainty/timestamp. On timeout/error: explicit saves fall back to SAVE_NEW; auto-signal saves skip write. **Not batch** — runs per-save, not periodically. |
| **Deduplication** | Not observed at core block level | LLM-prompt-driven: `add()` retrieves existing memories (`main.py:569–576`) and the update prompt decides ADD/UPDATE/DELETE actions (`main.py:591–598`) — no dedicated embedding-similarity dedup pipeline | autoDream consolidation phase deletes contradicted facts. openclaw: Light dreaming dedup at 0.9 similarity threshold | **Agent-based dedup** via `check_and_save()` (`_save.py:64–100`): save agent compares candidate against manifest of existing memories and decides SAVE_NEW vs UPDATE. On UPDATE: `overwrite_memory()` merges tags, refreshes metadata in-place. Article dedup: exact `origin_url` match via `_find_article_by_url()` (`articles.py:666–684`). |
| **Scoping** | Per-agent (agent_state carries block ownership via `blocks_agents` junction table, `blocks_agents.py:7–35`) | Per-entity: `user_id`, `agent_id`, `run_id` filters on all operations (`main.py:159–237`, enforced via `_build_filters_and_metadata()`) | Three-tier: user scope (cross-project), project scope (VCS), local scope (not VCS) (`agentMemory.ts:52–65`). openclaw: per-agent SQLite stores at `~/.local/share/openclaw/memory/{agentId}.sqlite` | **Two-tier scope**: memories are project-local (`.co-cli/memory/`); articles are user-scope cross-project (`~/.local/share/co-cli/library/`). FTS index (`co-cli-search.db`) is user-scope (indexes both). `search_knowledge` also indexes Obsidian and Google Drive sources when configured. No per-agent or per-user-id scoping within a scope. |
| **Injection** | Core blocks always in prompt (`prompt_generator.py:106–150`). Archival via tool call | External — caller retrieves and injects | Read at session start; referenced by model as needed. openclaw: hybrid vector+text search on query with temporal decay (30-day half-life), sync modes: session start, on-query, watch mode with 1.5s debounce | **Two injection layers**: (1) `add_always_on_memories()` (`agent.py:324–332`) — loads up to 5 memories with `always_on=True` every turn as standing context (`memory.py:269–281`). (2) `inject_opening_context()` history processor — FTS5/BM25 recall (max 3 results) on each new user turn, appended as `SystemPromptPart` (`_history.py:525–583`). Debounced by `MemoryRecallState` (`_types.py:12`) in session. Scoring: 0.6×relevance + 0.4×decay (exponential half-life, `memory.py:221–239`). Max chars: `memory_injection_max_chars` (default 2000). |
| **Cleanup** | Archival passages permanent | `delete_all()` available. Soft-delete via `is_deleted` flag | autoDream prunes during consolidation. openclaw: Deep dreaming auto-recovers below health threshold (0.35) | **Capacity-based retention** via `enforce_retention()` (`_retention.py:16–52`): when total > `memory_max_count` (default 200), deletes oldest non-protected entries (cut-only, deterministic). `decay_protected=True` entries exempt. FTS stale entries cleaned via `KnowledgeStore.remove_stale()` (`_store.py:1400–1446`). No TTL, no time-based expiry, no soft-delete. Deletion is permanent (file unlink). |

## 5. Extraction & Intelligence

| Aspect | letta | mem0 | fork-cc | co-cli |
|--------|-------|------|---------|--------|
| **Extraction method** | Manual — agent calls `memory()` tool explicitly | LLM-driven: `USER_MEMORY_EXTRACTION_PROMPT` extracts facts from messages (`prompts.py:62–120`). Returns `{"facts": [...]}` | Manual — model writes to memory files via tools. autoDream consolidation gathers signal from transcripts. openclaw: auto-saves session context via session-memory hook on `/new` or `/reset` (last 15 messages, LLM-generated filename slug) | **Hybrid**: (1) Manual — agent calls `save_memory` tool explicitly. (2) Auto-signal — `memory_auto_save_tags` config (default: `["user", "feedback", "project", "reference"]`, `config.py:193`) enables post-turn automatic saves for matching signal types at high confidence via `persist_memory(on_failure="skip")` (`_extractor.py:141–225`). Both paths share the same `persist_memory()` pipeline (agent-based upsert → write → retention). |
| **Update intelligence** | Manual str_replace — agent decides what to change | LLM decides add/update/delete via `DEFAULT_UPDATE_MEMORY_PROMPT` (`prompts.py:175+`). No dedicated embedding-similarity dedup — LLM prompt decides all operations | autoDream LLM: merge new signal, convert relative→absolute dates, delete contradicted facts (`consolidationPrompt.ts:10–65`). openclaw: Deep dreaming refines entries with recency decay + health threshold recovery | **Agent-based upsert**: save agent (`_save.py:64–100`) compares candidate against manifest of existing memories and decides SAVE_NEW vs UPDATE. | **Graph/relational** | Not observed | Optional: `MemoryGraph` in Neo4j.
