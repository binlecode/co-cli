## Context

3.5a shipped the no-LLM hygiene half: usage telemetry + manual user control (pin/unpin). 3.5b ports hermes's two autonomous mechanisms:

1. **Post-session combined review.** At session end, fork an agent scoped to skills + knowledge tools (`max_iterations=8`). Scans the just-finished conversation for user-correction patterns, drift the foreground model missed, and reusable procedures worth saving. Patches and creates skills/knowledge autonomously. Reports `💾 <summary>` to the user. Adapted from hermes `_COMBINED_REVIEW_PROMPT`.

2. **Curator (consolidation + state machine).** Pure state transitions (active → stale at 30 days unused → archived at 90 days). Forked agent scoped to skills toolset (`max_iterations=100`). Consolidates prefix-clustered narrow skills into class-level umbrellas. Trigger: REPL startup, idle-gated (default 2h), interval-gated (default 7 days between runs). Adapted from hermes `curator.py`.

What co-cli already has (Plan 3): in-session drift patching (the `## Drift` reflex in `06_skill_protocol.md` + `skill_manage(action='patch')`). 3.5b adds the post-session passes.

**Session-end firing rationale (v1).** Review fires inline in `_drain_and_cleanup`, immediately before the existing dream cycle. Session-end eliminates ~6 concurrency failure modes present in turn-boundary firing (single-in-flight check, REPL exit drain timeout, `asyncio.create_task` lifecycle, idle-tracking, etc.). Cost: long sessions get one extraction pass; unclean exits never review at all. Plan 3.5c converts to turn-boundary firing once v1 signal confirms single-pass review is insufficient.

## Problem & Outcome

**Problem.** Co-cli's skill library evolves only when the model notices something mid-turn (`## Drift` reflex) or when the user explicitly asks. Longer-horizon improvements — patterns that emerge across multi-turn sessions, drift the model didn't catch in-flight, deprecated skills cluttering the index — are never captured. The usage data 3.5a writes has no consumer.

**Failure cost.** Skill quality plateaus. The library grows monotonically. `skill_search` results degrade. User-driven improvements (corrections, style preferences) are absorbed into memory but not encoded into skills.

**Outcome.**

1. **Session-end combined review** — `_maybe_run_session_review(deps, message_history)` fires inside `_drain_and_cleanup` when `skills.review_enabled=True`. Forks an agent with `delegation="session_reviewer"` profile (skill AND knowledge tools), `max_iterations=8`. Reports `💾 <summary>` via `frontend.on_status`.

2. **Curator** — state machine + consolidation. Pure state transitions run at REPL startup if `(now - last_run > interval_hours) AND (idle_seconds >= curator_min_idle_hours * 3600)`. Stale at `> stale_after_days`; archived (file moved to `.archive/`) at `> archive_after_days`. Pinned skills opt out. After transitions, a forked agent runs consolidation — scoped to `delegation="skill_curator"`, `max_iterations=100` — merging prefix-clustered narrow skills into broader umbrellas. Per-run reports written as JSON (`run.json`) and markdown (`run.md`).

3. **CLI control surface.** `/skills curator status|run|pause|resume|restore <name>`, `/skills review run`.

4. **Protocol acknowledgment** in `06_skill_protocol.md` — short `## Background review` section telling the model the review and curator exist and that pinning is the opt-out.

## Behavioral Constraints

1. **Agent-created filter is absolute.** State transitions and curator consolidation apply only to skills where `is_agent_created(name, deps)` returns True. Bundled and URL-installed skills are upstream-managed.

2. **Pinned skills opt out.** `pinned: true` in the sidecar exempts a skill from all auto-state-transitions and curator consolidation.

3. **Archive is recoverable, not deleted.** Archived skills move to `~/.co-cli/skills/.archive/<name>.md`. Restoration via `/skills curator restore <name>`. The curator never deletes.

4. **Approval bypass is scoped to forked deps.** `auto_approve_skill_ops` and `auto_approve_knowledge_ops` default False on `CoRuntimeState`. Set True only inside `fork_deps_for_reviewer()` / `fork_deps_for_curator()` factories. The actual bypass mechanism is `requires_approval=False` at tool registration (agents/core.py:202) — pydantic-ai never fires the approval gate for delegation agents. The flags are the explicit, testable permission contract; the `requires_approval=False` is the mechanism. Foreground tool calls are unaffected.

5. **Review and curator are opt-in by default.** `skills.review_enabled: bool = False`, `skills.curator_enabled: bool = False`. Explicit user opt-in required for autonomous LLM mutation of the skill/knowledge library.

6. **Errors never propagate.** Review and curator failures call `logger.warning("...", exc_info=True)` and return. Pattern copied from `_maybe_run_dream_cycle`.

7. **Iteration budgets are bounded.** Review: `REVIEW_MAX_ITERATIONS=8`, `REVIEW_TIMEOUT_SECONDS=120`. Curator: `CURATOR_MAX_ITERATIONS=100`, `CURATOR_TIMEOUT_SECONDS=600`.

8. **Forks use the foreground model.** Same model object passed through `fork_deps`; same credentials, same provider, same tracing stream.

9. **Reviewer/curator prompts frame the model as a maintainer.** Includes hermes's "Be ACTIVE" framing ("most sessions produce at least one update; a pass that does nothing is a missed learning opportunity, not a neutral outcome") plus the 3-tier preference order: UPDATE LOADED skill → UPDATE EXISTING UMBRELLA → CREATE NEW UMBRELLA. Session-specific names (`fix-pr-123`) are forbidden — only class-level taxonomies. **Never call `skill_manage(action='delete')` or `knowledge_manage(action='delete')` from a fork.**

10. **Curator idle gating.** Curator triggers only when BOTH `(now - last_run_at) > curator_interval_hours` AND `(now - session.last_user_input_at) >= CURATOR_MIN_IDLE_HOURS * 3600`. On REPL startup, `last_user_input_at` is None — idle is effectively infinite; first run is unblocked.

11. **Per-run reports persist as JSON + markdown.** Both reviewer and curator write `run.json` + `run.md` to per-agent directories: `~/.co-cli/session-reviews/<timestamp>/` and `~/.co-cli/curator-runs/<timestamp>/`. Timestamp = `YYYYMMDD-HHMMSS` with `-<hash>` suffix on same-second collisions. `run_id` recorded inside `run.json`.

12. **Combined review handles skills and knowledge in one fork.** Single forked agent has BOTH skill and knowledge toolsets. Single pass updates both surfaces.

13. **Session review blocks REPL shutdown up to `REVIEW_TIMEOUT_SECONDS`.** Awaited inline in `_drain_and_cleanup` via `asyncio.wait_for`. 3.5c removes the blocking by converting to background firing.

14. **Compacted transcript is a v1 known limitation.** If compaction ran during the session, `state.message_history` passed to the reviewer is a summary+recent-turns list — the reviewer sees only post-compaction context. Accepted: user-correction signal lives in the most recent turns, which survive compaction verbatim. 3.5c (turn-boundary firing) eliminates this by reviewing incrementally.

## Failure Modes

1. **Reviewer over-creates narrow skills.** Mitigated by `SESSION_REVIEW_PROMPT` forbidding session-specific names; curator consolidates narrow siblings on the next pass.

2. **Curator merges skills the user values.** Mitigated by `pinned` opt-out; archive (recoverable) only, never delete; per-run reports for audit; `restore` command.

3. **Approval bypass leaks into foreground.** Mitigated by: flags live on `CoRuntimeState` which `fork_deps()` resets to defaults; True only inside named factories. Actual bypass is `requires_approval=False` at registration — applies only to delegation agents, never to the foreground toolset. Test: flags False after plain `fork_deps()`, True only after factory calls.

4. **Session review delays REPL shutdown.** Mitigated by `REVIEW_TIMEOUT_SECONDS=120` outer cap + `REVIEW_MAX_ITERATIONS=8` inner budget. 3.5c removes at-exit blocking.

5. **Sidecar corruption.** Mitigated by 3.5a's atomic writes; curator treats missing records as `state="active"` with `created_at=now()`.

6. **Curator blocks REPL startup.** Mitigated by `asyncio.create_task(...)` — spawned, not awaited; REPL interactive immediately.

7. **Curator fires while user is mid-task.** Mitigated by idle gating (`CURATOR_MIN_IDLE_HOURS=2`). On startup, idle is infinite so first run is unblocked.

8. **Reviewer patches a skill the foreground agent is about to use.** Not applicable in 3.5b — session-end review fires after the last foreground turn. 3.5c re-introduces this concern.

9. **Reviewer or curator hangs.** Mitigated by outer `asyncio.wait_for` with timeout; cancel on expiry.

10. **Multiple curator runs concurrently (two REPLs).** Mitigated by optimistic concurrency on state file: read state → check `should_run_now` → write back with new `last_run_at`; abort if on-disk timestamp differs from read value. Best-effort; collision probability is negligible given the 7-day default interval.

11. **Status callback surfaces while user is typing.** Review fires only at REPL exit. Curator fires at startup where `TerminalFrontend._status_live` buffering already handles it (display/core.py:282–285).

12. **Background curator task does sync setup before first `await`.** Mitigated by `await asyncio.sleep(0)` at the start of `maybe_run_curator`.

### Approval bypass for sub-agents

Two flag fields added to `CoRuntimeState` (must NOT be cleared by `reset_for_turn()`):
```python
auto_approve_skill_ops: bool = False
auto_approve_knowledge_ops: bool = False
background_status_callback: Callable[[str], None] | None = field(default=None, repr=False)
```

Fork factories set the flags:
```python
def fork_deps_for_reviewer(parent: CoDeps) -> CoDeps:
    child = fork_deps(parent)
    child.runtime.auto_approve_skill_ops = True
    child.runtime.auto_approve_knowledge_ops = True
    return child

def fork_deps_for_curator(parent: CoDeps) -> CoDeps:
    child = fork_deps(parent)
    child.runtime.auto_approve_skill_ops = True
    # knowledge ops stay gated — curator scope is skills-only
    return child
```

Actual bypass: `build_agent` delegation path registers all tools with `requires_approval=False` (agents/core.py:202). The flags are the testable permission contract; `requires_approval=False` is the mechanism. No modifications to `_skill_manage_approval_subject`, `_knowledge_manage_approval_subject`, or `ApprovalKindEnum` are needed.

`background_status_callback` is wired in `bootstrap/core.py:create_deps` once before return: `deps.runtime.background_status_callback = frontend.on_status`.

### Standalone agent runner

```python
# co_cli/agents/_runner.py
async def _run_agent_standalone(
    agent: Any,
    prompt: str,
    deps: CoDeps,
    budget: int,
    model_settings: Any,
    *,
    role: str,
) -> tuple[Any, RunUsage, str]:
    with _TRACER.start_as_current_span(role) as span:
        span.set_attribute("agent.role", role)
        span.set_attribute("agent.request_limit", budget)
        result = await agent.run(
            prompt,
            deps=deps,
            usage_limits=UsageLimits(request_limit=budget),
            model_settings=model_settings,
            metadata={"session_id": deps.session.session_path.stem[-8:], "role": role},
        )
        usage = result.usage()
        span.set_attribute("agent.requests_used", usage.requests)
        return result.output, copy(usage), result.run_id
```

### Session-end review

`_drain_and_cleanup` extended with `message_history: list[ModelMessage] | None = None`. Ordering inside:
1. Await background curator task (5s timeout, then cancel) — curator writes settle before review reads.
2. `_maybe_run_session_review` — extracts new skills + knowledge from this session's transcript.
3. `_maybe_run_dream_cycle` (unchanged) — consolidates knowledge artifacts, including any the review just wrote.

```python
async def _maybe_run_session_review(
    deps: CoDeps, message_history: list[ModelMessage]
) -> None:
    if not deps.config.skills.review_enabled:
        return
    from co_cli.agents.session_review import run_session_review
    from co_cli.config.skills import REVIEW_TIMEOUT_SECONDS
    logger = logging.getLogger(__name__)
    try:
        result = await asyncio.wait_for(
            run_session_review(deps, message_history),
            timeout=REVIEW_TIMEOUT_SECONDS,
        )
        cb = deps.runtime.background_status_callback
        if result.summary and cb is not None:
            cb(f"💾 {result.summary}")
    except asyncio.TimeoutError:
        logger.warning("Session review timed out")
    except Exception:
        logger.warning("Session review failed", exc_info=True)
```

`run_session_review()`:
```python
async def run_session_review(deps: CoDeps, message_history: list[ModelMessage]) -> SessionReviewResult:
    child_deps = fork_deps_for_reviewer(deps)
    agent = build_agent(
        config=deps.config,
        model=deps.model.model,
        instructions=SESSION_REVIEW_INSTRUCTIONS,
        tool_fns=discover_delegation_tools("session_reviewer", deps.config),
        output_type=SessionReviewOutput,
    )
    transcript = serialize_messages(
        message_history,
        deps.config.observability.redact_patterns,
        include_tool_results=False,
    )
    prompt = SESSION_REVIEW_PROMPT.format(transcript=transcript)
    output, usage, run_id = await _run_agent_standalone(
        agent=agent, prompt=prompt, deps=child_deps,
        budget=REVIEW_MAX_ITERATIONS, model_settings=deps.model.settings,
        role="session_review",
    )
    _write_review_report(deps, run_id, output, usage)
    return SessionReviewResult(summary=output.summary, run_id=run_id)
```

Curator state file (`~/.co-cli/skills/.curator_state.json`):
```json
{"version": 1, "last_run_at": "2026-05-12T14:48:40Z", "last_run_summary": "...", "run_count": 0, "paused": false}
```

Gate:
```python
def should_run_now(state, settings, now, idle_seconds, *, bypass_time_gate=False) -> bool:
    if not settings.curator_enabled: return False
    if state.get("paused"): return False
    if idle_seconds < CURATOR_MIN_IDLE_HOURS * 3600: return False
    if bypass_time_gate: return True
    last_run = state.get("last_run_at")
    if last_run is None: return True
    return (now - _parse_iso(last_run)) > timedelta(hours=settings.curator_interval_hours)
```

`_summarize_skill_inventory(deps) -> str` — builds the curator prompt's inventory view:
1. Load sidecar via `read_records(deps)`.
2. For each entry where `state != "archived"` and `pinned != True` and `is_agent_created(name, deps)`: read `deps.user_skills_dir / f"{name}.md"`; include name, state, use_count, last_used_at, and first 1000 chars of body.
3. Skills not tracked in the sidecar are excluded — curator only manages sidecar-tracked skills.

`maybe_run_curator(deps, *, bypass_time_gate=False)`:
1. `await asyncio.sleep(0)` — yield before any sync I/O (Failure Mode 12).
2. Read curator state; check `should_run_now`. Return early if gated.
3. **Phase 1:** `apply_state_transitions` → write sidecar → `archive_skill` for each archived transition.
4. **Phase 2:** Fork curator agent (`skill_curator` profile), run with `CURATOR_MAX_ITERATIONS` + `asyncio.wait_for(CURATOR_TIMEOUT_SECONDS)`.
5. **Phase 3:** Write `run.json` + `run.md`; update curator state (`last_run_at`, `last_run_summary`, `run_count`).

Phase 1 runs even if there are no agent-created skills. Errors in Phase 2 do not roll back Phase 1. All exceptions logged + swallowed.

`skill_curator` delegation profile: `{skill_search, skill_view, skill_manage}` — NO knowledge tools.

REPL startup wiring — two distinct insertion points:
1. `create_deps()` in `bootstrap/core.py`: `deps.runtime.background_status_callback = frontend.on_status` just before return.
2. `_chat_loop()` in `main.py`: after `deps` is ready, before `state = _IterationState(...)`:
```python
if deps.config.skills.curator_enabled:
    deps.session.background_curator_task = asyncio.create_task(maybe_run_curator(deps))
```
Also update `session.last_user_input_at = datetime.utcnow()` on each user input.

## Background review

A separate review agent runs at the end of each session (when enabled).
It scans the conversation for drift you missed in-flight, patterns
worth saving as new skills, and user-correction signals worth
memorializing as knowledge artifacts. Don't double up: focus the
