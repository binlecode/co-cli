# Plan: Todo Continuity

Task type: `code-feature`

## Context

Two functional gaps in co-cli's session-scoped todo system:

- **G1 — full-rewrite-only updates.** `todo_write` requires the model to resend the entire list to update one item. Real failure mode: model drops items mid-plan or burns tokens on list management instead of work.
- **G2 — `/resume` silently loses todos.** `co_cli/commands/resume.py` reloads message history but never restores `session.session_todos`. After resume, `todo_read` returns empty; the next compaction's `build_todo_snapshot` injects nothing. In-flight tasks silently de-prioritize across session boundaries.

## Problem & Outcome

**Problem:**
- G1 — `todo_write` is full-rewrite. For a 10-item plan, marking one step done re-sends all 10 items.
- G2 — `/resume` reloads `messages` via `load_transcript` but leaves `session.session_todos` at its default empty list.

**Failure cost:**
- G1 — model drops tasks mid-plan, sends stale snapshots after earlier items changed, or over-spends tokens on list management.
- G2 — user resumes a multi-turn workflow; model has no programmatic access to the in-flight todo list; the next compaction's `build_todo_snapshot` injects nothing; in-flight tasks silently de-prioritize.

**Outcome:**
- `todo_write(todos, merge=True)` updates named items by `id` without touching the rest. `merge=False` (default) replaces the full list.
- Every item carries a model-assigned `id`. Compaction format includes `id.` prefix so the model can reference items after compression.
- `/resume` rehydrates `session.session_todos` from the most recent `todo_write` `ToolReturnPart` found in loaded messages. No new storage artifact — the transcript stays canonical.

## Behavioral Constraints

**Item schema (after this delivery):**
```python
class TodoItem(TypedDict):
    id: str           # model-assigned, unique within session
    content: str
    status: Literal["pending", "in_progress", "completed", "cancelled"]
    priority: Literal["high", "medium", "low"]
```

**Validation contract — all-or-nothing for both modes:**
- Every item must have a non-empty `id`.
- Every item's `id` must be unique within the payload.
- `id` must not contain `.` (period) or whitespace — keeps the snapshot-fallback parser unambiguous (the snapshot uses `". "` as the id/content separator). Suggested form: short slug or numeric string.
- `content` must be non-empty after `.strip()` — whitespace-only content is rejected (applies to fresh write and merge updates).
- `status` ∈ `{"pending", "in_progress", "completed", "cancelled"}`.
- `priority` ∈ `{"high", "medium", "low"}`.
- If **any** item fails validation, return errors and apply **nothing**. `session.session_todos` is preserved unchanged.

**Fresh write (`merge=False`, default):**
- Every item must pass full validation: `id`, `content`, optional `status` (defaults to `"pending"` if absent), optional `priority` (defaults to `"medium"` if absent).
- On success: `session.session_todos = validated`.

**Merge (`merge=True`):**
- For each payload item, match `id` against existing `session.session_todos`:
  - **Known id:** update only fields explicitly present in the payload (`content`, `status`, `priority`). Validate each provided field; missing fields are left unchanged.
  - **Unknown id:** treat as a brand new item; must pass full fresh-write validation; appended to the end after existing items.
- Existing items not mentioned in the payload are preserved unchanged, in original order.
- New items (unknown ids) preserve their order from the payload.
- **`id` is the merge key and cannot be rewritten through merge.** To rename an id, do a fresh write.
- **Empty `content` in a merge update is a validation error** (same as fresh write).

**Tool return (both modes, on success):**
- `tool_output(..., todos=list(session.session_todos))` — the full post-state, as a list of `TodoItem` dicts. Resume hydration reads this metadata.

**Resume rehydration (`co_cli/commands/resume.py`):**
- After `load_transcript` returns messages, walk messages backwards looking for the **most recent of**:
  1. A `ModelRequest` containing a `ToolReturnPart` with `tool_name == "todo_write"` — read its `metadata['todos']` field (guard for `metadata is None`). This is the primary path.
  2. A `ModelRequest` containing a `UserPromptPart` whose content starts with `TODO_SNAPSHOT_PREFIX` — parse the snapshot lines (shape `- [{status}] {id}. {content}` after TASK-2) back into `TodoItem` dicts with `priority="medium"` (the snapshot strips priority). This is the **fallback for compacted sessions** where the `todo_write` `ToolReturnPart` was dropped during compaction.
- Whichever is found first (most recent) wins. Assign the result to `ctx.deps.session.session_todos`.
- **Defensive filter:** drop any rehydrated item whose `id` is missing/empty (defensive against pre-delivery transcripts whose `todo_write` metadata predates the `id` field).
- **Primary-vs-fallback divergence (documented, not a bug):** the primary path (`metadata['todos']`) restores **full state** including completed/cancelled items, in original order, with no cap — `todo_write`'s metadata carries `list(session.session_todos)` post-state. The fallback path restores **only active items** (pending/in_progress) and caps at **10 items**, because `_active_todos` filters terminal items and `_format_active_todos` truncates to 10 before emitting the snapshot. Completed/cancelled items lost to compaction are not recoverable from the snapshot — accepted because the active-items contract is what continues work; historical completions are reference, not load-bearing.
- If neither is found, leave `session.session_todos` at its default empty list.
- Rehydration runs after `session.session_path` is set, before `ReplaceTranscript` is returned.

**Compaction format:**
- `_format_active_todos` renders `f"- [{t['status']}] {t['id']}. {t['content']}"`.
- Both `_gather_session_todos` and `build_todo_snapshot` inherit the change automatically.

**Merge application** — preserve order of existing items, append new ones:
```python
existing_by_id = {t["id"]: t for t in session.session_todos}
existing_order = list(session.session_todos)

# Step A: partition validated payload into updates (known ids) vs new_items (unknown ids).
# `validated_payload` is the output of the validation pass — all-or-nothing has
# already gated us here, so every entry is well-formed.
updates: dict[str, dict] = {}        # id -> validated partial-update dict (id + provided fields)
new_items: list[TodoItem] = []       # full TodoItem dicts from fresh-item validation
for payload_item in validated_payload:
    if payload_item["id"] in existing_by_id:
        updates[payload_item["id"]] = payload_item
    else:
        new_items.append(payload_item)

# Step B: rebuild list preserving order of existing items, then append new items.
merged: list[TodoItem] = []
for t in existing_order:
    if t["id"] in updates:
        merged.append({**t, **updates[t["id"]]})
    else:
        merged.append(t)
merged.extend(new_items)
session.session_todos = merged
```

**Resume rehydration scan:**
```python
# co_cli/commands/resume.py
import re
from pydantic_ai.messages import ModelMessage, ToolReturnPart, UserPromptPart
from co_cli.context._compaction_markers import TODO_SNAPSHOT_PREFIX
from co_cli.deps import TodoItem

# Inline the canonical status set rather than importing `_VALID_STATUS` from
# `co_cli.tools.todo.rw` — that's a different package and underscore-prefixed
# module-level names are package-private. Mirrors the Behavioral Constraints
# "Validation contract" status enumeration.
_SNAPSHOT_VALID_STATUS = {"pending", "in_progress", "completed", "cancelled"}

# Tightened to forbid whitespace and period in the id capture, matching the
# id-charset rule in Behavioral Constraints.
_SNAPSHOT_LINE_RE = re.compile(r"^- \[(\w+)\] ([^.\s]+)\. (.+)$")


def _rehydrate_todos(messages: list[ModelMessage]) -> list[TodoItem]:
    """Find the most recent todo state in messages.

    Primary: a todo_write ToolReturnPart's metadata['todos'].
    Fallback: a TODO_SNAPSHOT_PREFIX UserPromptPart (compacted sessions).
    Whichever appears more recently wins. Returns [] if neither is found.

    Defensive: drops any rehydrated item without a non-empty `id`
    (guards against pre-delivery transcripts that predate the `id` field).
    """
    for msg in reversed(messages):
        for part in getattr(msg, "parts", []):
            # Primary path: most recent todo_write tool return
            if isinstance(part, ToolReturnPart) and part.tool_name == "todo_write":
                meta = getattr(part, "metadata", None)
                todos = meta.get("todos") if isinstance(meta, dict) else None
                if isinstance(todos, list):
                    return [
                        t for t in todos
                        if isinstance(t, dict) and str(t.get("id", "")).strip()
                    ]
            # Fallback: post-compaction snapshot
            if isinstance(part, UserPromptPart):
                content = getattr(part, "content", "")
                if isinstance(content, str) and content.startswith(TODO_SNAPSHOT_PREFIX):
                    return _parse_snapshot_lines(content)
    return []


def _parse_snapshot_lines(snapshot: str) -> list[TodoItem]:
    """Parse the body of a TODO_SNAPSHOT_PREFIX message into TodoItem dicts.

    Each active line has the shape (after TASK-2): `- [{status}] {id}. {content}`.
    Lines that don't match, have invalid status, or are missing id are skipped.
    Priority is not in the snapshot; defaults to 'medium'.
    """
    items: list[TodoItem] = []
    for line in snapshot.splitlines()[1:]:  # skip the prefix line
        m = _SNAPSHOT_LINE_RE.match(line)
        if not m:
            continue
        status, id_, content = m.group(1), m.group(2).strip(), m.group(3).strip()
        if not id_ or status not in _SNAPSHOT_VALID_STATUS:
            continue
        items.append({"id": id_, "content": content, "status": status, "priority": "medium"})
    return items
```
