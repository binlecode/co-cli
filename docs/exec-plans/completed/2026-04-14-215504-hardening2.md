## Behavioral Constraints

1. **Rename must be additive-then-remove.** All co-cli callers of `list_directory`, `find_in_files`, and `edit_file` must be updated in the same delivery. No deprecated aliases.
2. **patch fuzzy strategies must not silently mangle code.** Each strategy transforms search and file identically; the replacement is applied verbatim to the original (untransformed) file content. If no strategy matches exactly once, return an error — do not apply a fuzzy match that has multiple candidates.
3. **DENY additions must not catch legitimate developer operations.** `git reset --hard`, `git push --force`, `chmod`, `rm -rf <dir>` stay at REQUIRE_APPROVAL. Only remote-exec and fork-bomb patterns are added to DENY.
4. **Transcript failure must not abort the turn.** `_finalize_turn` catches `OSError` from `persist_session_history`, emits `frontend.on_status(...)`, and continues — the turn result is still returned.
5. **Skill failure must name the file.** `frontend.on_status(f"Skill {path.name!r} skipped: {e}")` — consistent with MCP failure pattern in `bootstrap/core.py:261`.
6. **No test skips. Real CoDeps, no mocks.**
7. **All tasks touch ≤5 files.** Exception: TASK-1 rename touches 7 files (inevitable for a system-wide rename); split acceptable given they are all targeted string replacements.

### TASK-2: Implement patch tool (fuzzy edit)

Replace `edit_file(path, search, replacement, replace_all)` with `patch(path, old_string, new_string, replace_all)` — same parameters renamed, same resource lock and staleness guard, fuzzy matching added.

**Four-strategy fallback chain** (tried in order, applied to both `old_string` and file content):

```
1. exact           — direct string match (current behavior)
2. line-trimmed    — strip leading/trailing whitespace per line before comparing
3. indent-stripped — strip all leading whitespace per line (tab/space drift)
4. escape-expanded — replace literal \n \t \r sequences with actual characters
```

Each strategy:
- Transforms `old_string` and `content` identically
- Locates the match position in the transformed content
- Maps position back to original content (character offset)
- Applies `new_string` (verbatim, untransformed) at that position in the original

If a strategy finds 0 matches → try next strategy.
If a strategy finds >1 match AND `replace_all=False` → return error (same as exact-match today).
If all strategies fail → return error naming the first strategy tried.

The position mapping (transformed offset → original offset) is done by tracking cumulative deletions from the transform. This preserves all surrounding whitespace and ensures the output is not mangled.

Register `patch` with `approval=True`, `sequential=True`, `retries=1` (same as `edit_file`). Remove `edit_file` registration.

### TASK-4: Shell DENY expansion

Add to `_shell_policy.py` before the ALLOW tier check:

```python
# Remote code execution via pipe-to-shell
if re.search(r'\b(curl|wget)\b.*\|\s*(ba|da|z|fi|k|c|t)?sh\b', cmd):
    return ShellPolicyResult(ShellDecisionEnum.DENY, "remote exec pattern (curl/wget pipe to shell)")

# eval with remote fetch
if re.search(r'\beval\b.*\$\(\s*(curl|wget)\b', cmd):
    return ShellPolicyResult(ShellDecisionEnum.DENY, "remote exec pattern (eval with curl/wget)")

# Fork bomb
if re.search(r':\s*\(\s*\)\s*\{.*:\s*\|.*:\s*&', cmd):
    return ShellPolicyResult(ShellDecisionEnum.DENY, "fork bomb pattern")
```

### TASK-5: Transcript write failure notification

Current: `append_messages`, `write_compact_boundary`, `write_session_meta` in `transcript.py` catch `OSError` and log WARNING. `persist_session_history` returns normally. Caller (`_finalize_turn`) has no signal.

Fix — two-part:
1. In `transcript.py`: remove the `try/except OSError` blocks, let exceptions propagate. The `logger.warning` calls become unnecessary (the caller will surface them).
2. In `main.py:_finalize_turn`: wrap the `persist_session_history` call:

```python
try:
    deps.session.session_path = persist_session_history(...)
except OSError as e:
    frontend.on_status(
        f"Session write failed — conversation may not be saved. Check disk space. ({e})"
    )
```

### TASK-6: Skill load actionable diagnostics

Current: `_load_skill_file` catches all exceptions with `logger.warning(f"Failed to load skill {path}: {e}")`. Frontend is not available inside `_load_skills`.

Fix — accumulator pattern (backward-compatible, no signature break at most call sites):
1. Add `errors: list[str] | None = None` parameter to `_load_skill_file` and `_load_skills`.
2. In `_load_skill_file` except block: append `f"Skill {path.name!r} skipped: {e}"` to `errors` if not None (keep `logger.warning` for file-based log).
3. In `bootstrap/core.py` call site: pass `errors=[]`, then iterate and call `frontend.on_status(msg)` for each.
4. In `_commands.py` `/skills reload` call site: same pattern using the REPL's frontend.
