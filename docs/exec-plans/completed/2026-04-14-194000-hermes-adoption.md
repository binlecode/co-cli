# Plan: hermes-adoption

## Context
co-cli currently uses distinct tools for similar operations (`list_directory` vs `find_in_files`) and strict exact-match editing (`edit_file`). This creates friction: LLMs struggle with whitespace in exact-match edits, and having non-standard names (`list_directory`, `find_in_files`) increases cognitive load. hermes-agent has solved these issues with standard-named tools and fuzzy patching.

## Problem & Outcome
Problem: Current tool surface uses non-standard names and rigid exact-matching for edits.
Failure cost: Increased tool errors due to non-standard names, and repeated failed turns when LLMs slightly botch whitespace in `edit_file` exact matches.
Outcome: A simpler, more resilient tool surface with standard Unix-like names (`glob`, `grep`) and drastically improved code editing reliability via fuzzy patching.

## Scope
- Tool renaming: rename `list_directory` to `glob` and `find_in_files` to `grep` to align with standard LLM expectations.
- Improve editing: port hermes-agent's fuzzy patch logic and replace `edit_file` with `patch(path, old_string, new_string)`.
- Update references: ensure all skills and docs reference the new tool names.

## Behavioral Constraints
- `glob` and `grep` must maintain the exact return output structure of their predecessors.
- `patch` must safely fallback through whitespace/indentation relaxations without silently mangling code. It should hold the resource lock like `edit_file` does.
- All mutating tools (`patch`) must be registered with `approval=True`.
- The `_native_toolset.py` must be updated to register the new tools and drop the deprecated ones.

## High-Level Design
1. **glob and grep tools**: Rename the functions and their tool descriptions in `co_cli/tools/files.py`. Update parameter documentation.
2. **patch tool**: Implement a set of fallback matching strategies (exact match, normalize newlines, normalize indents, strip all whitespace) ported from hermes-agent to locate and replace `old_string` even if LLM whitespace is slightly off. Preserves `replace_all` functionality. Register it with `approval=True`.
