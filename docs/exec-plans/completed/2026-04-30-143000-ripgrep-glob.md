# Plan: Ripgrep-based File Globbing

## Context
The `file_find` tool currently relies on Python's `pathlib.Path.glob()` for resolving recursive file patterns (e.g., `**/*.py`). While accurate, `Path.glob()` traverses all directories, including notoriously massive, ignored directories like `node_modules/`, `.venv/`, and `.git/`. This causes significant latency when exploring large codebases. The `file_search` (grep) tool already utilizes `ripgrep` (`rg`), which is extremely fast and natively respects `.gitignore`. By aligning `file_find` to use `rg` for globbing, we can achieve a critical performance leap for the coding agent.

## Problem & Outcome
**Problem:** Recursive file searches (`file_find` with `**/*`) stall the agent for multiple seconds on mid-to-large repositories because it scans ignored dependencies and build artifacts.
**Failure cost:** High latency leads to delayed agent responses, unnecessary token consumption (if the agent assumes a search failed), and poor user experience when navigating standard codebases.
**Outcome:** `file_find` executes almost instantly, leveraging `rg` to prune ignored directories natively, matching the speed of the existing `file_search` tool.

## Scope
- Refactor `co_cli/tools/files/read.py` to use a new asynchronous `_glob_ripgrep` function for recursive patterns.
- Fallback to the existing `_glob_python` implementation if `rg` is not installed or if it fails.
- Maintain the current output format (JSON objects with `name` and `type`) and the `mtime`-based sorting (newest first). Note: Recursive `rg` queries will only return files, not directories.

## Behavioral Constraints
- Must respect `.gitignore` natively (via `ripgrep`).
- Must continue to sort the matched files by modification time (`mtime`), newest first.
- Must gracefully fall back to Python `glob` if `ripgrep` is unavailable.
- Non-recursive queries (e.g., `pattern="*"`) should continue using the fast `iterdir()` approach without spawning `rg`.
- Recursive `rg` searches will exclusively return files, not directories. Update the `file_find` tool docstring to inform the LLM of this behavior.

## High-Level Design
1. **Ripgrep Invocation:** Introduce `_glob_ripgrep(resolved: Path, workspace_root: Path, pattern: str, max_entries: int)`.
   - Execute: `rg --files --null --hidden -g <pattern>` in the `resolved` directory.
   - Using `--null` allows safe splitting of file paths containing spaces.
   - `rg` naturally ignores `.git/` and `.gitignore` matches even with `--hidden`.
   - Update `file_find` docstring: "Note: recursive file searches (e.g., `**/*`) are highly optimized and will only return files, not directories."
2. **Parsing & Sorting:** 
   - Parse the null-separated byte output.
   - Run `_safe_mtime` across all matches using `asyncio.to_thread` to avoid blocking the async event loop with thousands of synchronous `os.stat` calls.
   - Sort all matches newest-first, then map them to the required dictionary structure: `{"name": str, "type": "file"}`.
   - Truncate to `max_entries` *after* sorting.
3. **Integration:** Update `file_find` to check `_has_command("rg")` and route to `_glob_ripgrep` for recursive patterns; fallback to `_glob_python` otherwise.
