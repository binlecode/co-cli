# Capability Enhancement ‚Äî vision input (`image_view`)

Task type: code

## Context

A capability enhancement that extends co's surface in a direction it cannot
currently reach, sourced from the cross-review against hermes-agent
(`docs/reference/RESEARCH-tools-gaps-co-vs-hermes.md` ¬ß2.1, ¬ß5). It is
**additive** ‚Äî it does not fix a regression; it opens a new mode on top of
existing, proven plumbing:

- **Vision input** (`image_view`): co has no way to feed pixels to a model. It can
  read a PDF's text (documents skill) but cannot look at a screenshot, photo,
  diagram, or rendered chart. This is greenfield ‚Äî a repo-wide grep confirms **no
  `BinaryContent` / `ImageUrl` / multimodal user-content** is constructed anywhere
  in `co_cli/` today.

> `2026-05-28-200025-toolgap-interactive-terminal.md` (Phase 2), where it sits

> vision-only.

### Why it rides proven infrastructure

- **Vision** reuses the `judge_model` precedent: `LlmSettings` already carries an
  optional pinned secondary model name (`judge_model: str | None`) that inherits
  provider/host/api_key. Vision adds the same shape for an optional vision model.

### Verified current state (2026-05-28)

- `build_model(llm)` (`co_cli/llm/factory.py:34`) returns `LlmModel(model,
  settings, settings_noreason)`. `ollama` ‚Üí `OpenAIChatModel` (vision works only
  if the configured model is vision-capable, e.g. `qwen2.5-vl`; the default
  agentic text model is not). `gemini` ‚Üí `GoogleModel` (natively multimodal).
- `LlmSettings` (`co_cli/config/llm.py:199`) has the optional pinned
  `judge_model: str | None` ‚Äî the exact precedent for an optional secondary model.
- `pydantic_ai.messages.BinaryContent(data, media_type)` is the provider-agnostic
  way to attach a local image; both backends accept it. A description call is a
  **single** multimodal request, not a tool-using sub-agent.

### Hermes parity reference (grounded, not copied)

- **Vision** ‚Äî `hermes-agent/tools/vision_tools.py` exposes `vision_analyze`
  against its configured multimodal backend. co cannot copy the backend wiring
  (co's model is a pydantic-ai object); what transfers is the **tool shape**:
  local image + question ‚Üí text answer.

## Problem & Outcome

**Problem.** Any image input has nowhere to go ‚Äî `file_read` rejects binary and no
path feeds pixels to a model.

**Outcome.** **`image_view(path, prompt)`** reads a local image, attaches it as
`BinaryContent`, runs a single multimodal request, returns the model's text
answer. Capability-gated: usable only when a vision-capable model resolves
(Gemini agent model, or Ollama with an optional `llm.vision_model`); the
`check_fn` reports unavailable rather than sending images to a text model that
would hallucinate.

  `build_judge_model`.
- Bootstrap (`co_cli/bootstrap/`) + `co_cli/deps.py` ‚Äî resolve once onto
  `deps.vision_model: LlmModel | None`, parallel to `deps.model`.
- `image_view` `check_fn` gating on `deps.vision_model is not None`.
- `docs/specs/tools.md` ‚Äî documented with caveats.
- Tests: `tests/test_flow_vision.py` (new).

### Out of scope
- **Video / screenshot capture / screen control / image generation** ‚Äî
  `computer_use`, `video_analyze`, `image_generate` stay out (research doc ¬ß2.2).
- **Remote image URLs** ‚Äî `image_view` reads local paths only (`url` arg is a
  trivial follow-up; callers can already `web_fetch` text).
- **A vision sub-agent** with its own tool loop ‚Äî single model request, no
  `TaskAgentSpec`. No auto-swapping the main agent model to multimodal mid-turn.

## Behavioural Constraints
1. **Honest capability gate.** On Ollama with no `vision_model`, `image_view` is
   unavailable (`check_fn` false) with a remediation message. It must **never**
   send image bytes to a text-only model and present the result as if the model
   saw the image.
2. **Single request, no agent loop.** One multimodal request per call, bounded by
   the same per-call timeout discipline as other model calls.
3. **Local-file boundary.** `path` resolved + workspace-confined like `file_read`;
   reject escapes.
4. **Size + media-type guards.** Cap at ~20 MB; accept only
   `image/{png,jpeg,webp,gif}`; reject others with a clear error (PDFs ‚Üí documents
   skill).
5. **No new dependency** ‚Äî pydantic-ai's existing `BinaryContent` only.

## High-Level Design

### A1 ‚Äî `build_vision_model` (factory)
```python
def build_vision_model(llm: LlmSettings) -> LlmModel | None:
    if llm.uses_gemini():
        return build_model(llm)                       # natively multimodal
    if llm.uses_ollama() and llm.vision_model:
        return build_model(replace(llm, model=llm.vision_model))
    return None                                       # Ollama, no vision_model
```
