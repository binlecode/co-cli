# Split `_history.py` into `_compaction.py` + `_prompt_text.py`

## Context

`co_cli/context/_history.py` is 1277 lines and currently hosts three orthogonal responsibilities:

1. **Compaction subsystem** (lines 1-1107) — history processors (dedup, truncate, enforce_batch_budget, summarize_history_window), the shared planner (plan_compaction_boundaries, markers, group_by_turn), the hygiene and overflow-recovery entry points, and the semantic markers / prefixes.
2. **Dynamic-instruction text** (lines 1120-1277) — `_recall_prompt_text` (per-turn personality + knowledge recall) and `_safety_prompt_text` (per-turn doom-loop / shell-error warnings). These are registered via `agent.instructions()` and never appear in `message_history`.

These two subsystems do not share state and only overlap incidentally (both happen to scan messages for the last user turn). Splitting clarifies ownership, shrinks the surface of the compaction module to just compaction, and lets specs point at a file that represents exactly what it claims to.

This is the deferred item from the 2026-04-23 compaction-cleanup pass (plan: `docs/exec-plans/completed/2026-04-23-143810-compaction-todo-snapshot.md` after ship). It is a pure refactor — **no behaviour changes**.

A focused check: `rg "co_cli\.context\._history|from co_cli\.context\._history|context\._history" co_cli tests evals docs` must be empty.

## Optional: split `tests/test_history.py`

`tests/test_history.py` (~1000 lines per memory from recent work) mixes compaction planner tests, `/compact` dispatch tests, and recall / safety-prompt tests. It is the natural counterpart to split:

- `tests/test_compaction.py` — planner, markers, hygiene, overflow, processor tests.
- `tests/test_prompt_text.py` — recall prompt text, safety prompt text, doom-loop / shell-error counters.

**Decision point.** This is mechanical but not coupled to the src split. Leaving it as `test_history.py` is valid for a focused diff; splitting is cleaner and should be a single follow-up commit. Recommend splitting **in the same PR** to keep filename-to-subject pairing consistent — it is cheaper to split tests once than to untangle them later.
