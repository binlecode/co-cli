# Documents Skill — Skill Over `document_extract`

Task type: skill content (no new tool code)

## Context

This plan authors the **`documents` skill** — the locate → extract →
answer-over-local-documents workflow. It is the **skill half** split out of the
deleted `2026-05-27-150940-tool-document-handling.md`; the **tool half**
(`document_extract`) is the batch-2 tool-gap plan
(`2026-05-27-172717-toolgap-b2-document-extract.md`, its TASK-2).

**Why a separate plan.** A skill is a prompt overlay over tools; a skill over a
missing tool is inert. The substrate gate (`.agent_docs` skill rules; mirrored
in the skill-porting plan) blocks a `documents` skill until its tool exists. So
this plan is **hard-gated** on `document_extract` landing green.

**Mission + convergence.** Local document handling is mission-core for a
personal assistant and the highest-convergence capability co lacked — 2 of 3
surveyed peers ship it (`RESEARCH-skills-peers-tiers.md` §2.5 T2-E4: hermes
`ocr-and-documents`+`nano-pdf`, openclaw `nano-pdf`).

## Problem & Outcome

**Problem.** Even once `document_extract` ships, the model has no *workflow*
guidance for "find the user's PDF, pull its text, answer with page citations."
A bare tool without a skill leaves the orchestration to chance.

**Outcome.**
1. **`co_cli/skills/documents.md`** — §6-shaped workflow skill over
   `document_extract` (+ `file_find`/`file_search`/`google_drive_search` for
   locate).
2. **Bundled-library gate updated** — count 6 → 7, `documents` added to the
   expected-names set.

## Behavioural Constraints
1. **Hard substrate gate.** Do not author until `document_extract` (tool-gaps
   plan TASK-4) is green; the skill names the exact tool it drives.
2. **Read-only synthesis.** Locate → extract → answer; the skill mutates
   nothing (no write tools in the workflow).
3. **Cite page/section.** Answers reference the page or section the text came
   from, leveraging `document_extract`'s preserved page breaks.
4. **Lint-clean before ship.** R1–R4 + B1 clean, security scan empty, body
   < 8000 chars, loads via `/skills reload`.

## High-Level Design
§6-shaped workflow body:
- `## Phase 1 — Locate`: find the document (`file_find`/`file_search`/
  `google_drive_search`/user-supplied path).
- `## Phase 2 — Extract`: `document_extract(path)`.
- `## Phase 3 — Answer`: summarize / answer the user's question with citations
  to page/section. Read-only synthesis.
