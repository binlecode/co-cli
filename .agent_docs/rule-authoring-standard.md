# Rule Authoring Standard — Low-Inference Reflexes

The behavioral rules in `co_cli/context/rules/*.md` steer a **weak configured model** that under-executes high-inference judgment calls. Write each rule as a **low-inference reflex**, not a metacognitive ask. A rule is a reflex when it: (1) fires on an **observable cue self-evident at the moment it fires** ("before you ask the user X", "when a tool returns an error", "after two same-method failures") — never "when you suspect…", "if it seems relevant…", or anything requiring the model to know what it does not know; (2) states **one imperative action**, not a paragraph of considerations to weigh; (3) **names the concrete tool** in plain words (`memory_search`) — never the category ("search tools") and never with `tool_name(` call syntax (trips the F5 floor guard); (4) **shows the anti-pattern in quotes** where a wrong behavior is common ("don't restate an answer you already gave"); (5) **enumerates the exceptions up front** so the residual default needs no judgment; (6) is **co-fit, not coding-fit** — borrow the reflex *form* from terse coding prompts but not their length caps; co is knowledge-work and reflexes brake redundant/no-op steps, never thoroughness-during-work. This is the yardstick for every future rule edit and the anti-erosion lever: without it, rule style gets re-litigated every edit. When a rule fails one of these, it is a judgment-call candidate for a future evidence-gated reflex rewrite — rewrite only with a demonstrated failure or a clean ablation signal, never on faith.
