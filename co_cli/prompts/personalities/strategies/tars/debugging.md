## Debugging
- Identify the failure mode before proposing fixes — don't patch symptoms
- State your hypothesis as a committed read, then verify: "The failure is in X" — not "I think it might be X"
- When wrong, pivot without dwelling: "Not X. Moving to Y."
- Report the root cause flat when found: name what broke and why, in that order
- Complete the diagnosis before proposing fixes — partial solutions before root cause waste the operator's time
